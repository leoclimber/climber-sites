# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Climber Sites is a tool that generates premium, single-page marketing websites for local businesses (mostly Dublin-area: burger joints, cafés, barbershops, salons, etc.). It's a single static `index.html` frontend (vanilla JS, no framework/bundler) backed by Vercel serverless functions in `api/`. There is no build step — this deploys to Vercel as-is.

Core loop, driven from `index.html`:
1. **Find clients** (`api/find-leads.js`) — search Google Places for businesses of a type in a city, flag which ones have no website (best leads).
2. **Extract business data** (`api/extract-business.js`) — paste a Google Maps / Instagram URL, resolve it to a Place ID, pull real business details/photos/reviews from Google Places, then have Claude enrich it into a structured content brief (headline, vibe, services, colors, fonts).
3. **Generate the site** (`api/generate-site.js`) — the core endpoint. Takes the business brief + a "mode" and streams back a full single-file HTML/CSS/JS site generated by Claude.
4. **Publish** (`api/publish-site.js`) — pushes the generated HTML straight to a new Vercel deployment via the Vercel API and returns a public `.vercel.app` URL.
5. **Attach domain** (`api/attach-domain.js`) — attaches a client's custom domain to that Vercel project and returns DNS instructions to hand to the client.

## Commands

There is no package manager setup, build, lint, or test tooling in this repo (`package.json` has no scripts/dependencies). Development is done by editing `index.html` and the files under `api/` directly.

- Run locally: `vercel dev` (uses `vercel.json` function config) — requires the Vercel CLI and the env vars below.
- Deploy: push to the connected Git remote; Vercel auto-deploys `main` (see recent commit history — deploys happen via direct commits/uploads to `main`, no PR workflow currently).

### Required environment variables (set in Vercel project settings, not in the repo)
- `ANTHROPIC_API_KEY` — used by `generate-site.js` and `extract-business.js`
- `GOOGLE_PLACES_API_KEY` — used by `find-leads.js` and `extract-business.js`
- `VERCEL_TOKEN` — used by `publish-site.js` and `attach-domain.js` to deploy client sites via the Vercel API
- `VERCEL_TEAM_ID` — optional, only needed if the Vercel account is a team

## Architecture

### `api/generate-site.js` — the core generation engine
This is a large (~2500 line) single file; almost all the product's value lives here. Key structure:

- **Categorization**: `businessType` free text is matched against keyword lists to bucket the business into a `category` (burger, cafe, pizza, barber, salon, gym, dental, etc. — see the big if/else chain near the top of the handler). Category drives which art-direction "kit", image bank, and prompt sections get used.
- **Art direction kits** (`kits` object): per-category palette, fonts, and tone/signature guidance that gets woven into the prompt sent to Claude.
- **Modes** — 3-slot system per category, passed in as `body.mode`:
  - `clean` — sober/elegant, default, every niche supports it
  - `cinematic_a` / `cinematic_b` — niche-specific cinematic variants with bespoke scroll-driven effects (legacy value `"cinematic"` is coerced to `cinematic_a`)
- **Image/video bank**: stock photos/videos per category live in `banco/<category>/` in this repo and are served from GitHub raw (`https://raw.githubusercontent.com/leoclimber/climber-sites/main/banco/<category>/...`) — see `BANK_REPO`/`bankBase`. Client-supplied photos (via `clientPhotos`) are preferred; the bank is only used as filler when the client provides fewer than 4 real photos.
- **Stamped showpiece components**: some cinematic effects (e.g. `buildPourA` — the café "scroll-scrub" video hero) are NOT generated by the LLM. They're pre-built, deterministic HTML/CSS/JS blocks that the prompt tells Claude to leave a placeholder for (e.g. `THE_POUR_A_PLACEHOLDER`), and the server string-replaces the placeholder with the real block after streaming completes. This guarantees these specific effects are pixel/behavior-identical every generation instead of relying on the LLM to reproduce fiddly scroll-sync JS. The same pattern is used for injecting the client's real hero video (`HERO_VIDEO_SRC` placeholder swapped for the actual base64/URL) to avoid sending giant base64 blobs through the prompt.
- **Edit mode**: if `editInstruction` + `previousHtml` are both present, the request is treated as an edit of an existing generated site rather than a fresh build, and a cheaper/faster model is used.
- **Model selection**: `claude-opus-4-8` for fresh generation, `claude-sonnet-4-6` for edits (see `useModel`). Generation is streamed (`stream: true`) from the Anthropic Messages API directly (no SDK) and relayed to the client as it arrives, with a 10s heartbeat write to keep the Vercel function connection alive during long generations (`maxDuration: 260` in `vercel.json`).
- **Response protocol quirk**: once `res.writeHead` has been called to start streaming (`headersSent = true`), no code path may call `res.status().json()` again — it must fall back to `res.write()`/`res.end()` on error, or Node throws `ERR_HTTP_HEADERS_SENT`. Preserve this pattern when touching error handling in this handler.

### Other API endpoints
- `api/extract-business.js` — resolves a Google Maps URL to a Place ID through several fallback strategies (direct `place_id` param, coordinates + name via nearby/text search, short-link redirect resolution), fetches Place Details, then calls Claude once to turn raw business data into a structured content brief (JSON) consumed by the generator.
- `api/find-leads.js` — Google Places Text Search for `"<businessType> in <city>"`, then Place Details per result to check for a website; can filter to only businesses without one.
- `api/publish-site.js` — creates a Vercel deployment with the generated `index.html` inlined directly in the request body (no repo needed per client site); sanitizes the project name for Vercel's naming rules.
- `api/attach-domain.js` — adds a custom domain to a Vercel project and returns the A/CNAME records the client needs to configure at their registrar.

All endpoints follow the same shape: manual CORS headers (`Access-Control-Allow-Origin: *`), OPTIONS preflight short-circuit, POST-only, and a top-level try/catch returning `{ error, detail }` on failure. There's no shared middleware/utils file — this boilerplate is duplicated in each handler.

### `index.html` — the operator UI
Single HTML file with inline `<style>` and `<script>`, no framework. Two tabs: "Gerar Site" (the generation form + live iframe preview + publish/attach-domain actions) and "Achar Clientes" (the lead finder). Talks to the API endpoints above via plain `fetch`. Portuguese is used for UI copy/comments; generated site copy itself follows the business's locale (English for Ireland, Portuguese otherwise — see the language rule near the end of the generation prompt).

### `banco/` — per-category stock asset bank
Each subdirectory (`burger/`, `cafe/`, ...) holds the stock photos/videos referenced by `bankBase` in `generate-site.js`. Filenames are a fixed contract the generator code depends on (`hero.jpg`, `1.jpg`–`4.jpg`, `ambiente.jpg`, plus category-specific videos like `stack.mp4`, `sear.mp4`, `pour.mp4`, `pour_scene.mp4`). Adding a new category means adding a new `banco/<category>/` folder with these same filenames, a matching entry in the `kits` object, and categorization keywords in the if/else chain.
